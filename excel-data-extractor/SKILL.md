# Excel 数据提取技能

根据用户要求，从Excel文件中智能提取和汇总数据。专为农业补贴数据处理（产补/地力补贴）优化，支持 寨隆镇 等乡镇级补贴核验场景。

---

## ⛔ 禁止事项 (Anti-Pattern Blacklist)

以下操作在本技能中 **严格禁止**：

| # | ⛔ 禁止行为 | 原因 | 正确做法 |
|---|-----------|------|---------|
| 1 | **修改源Excel文件** | 原始数据不可逆损坏 | 只读读取 → 结果写入新文件 |
| 2 | **跳过结构分析直接读数据** | 列名/Sheet名未知导致读取失败 | 先 `analyzeExcelStructure` 再 `readDataBySheetName` |
| 3 | **硬编码列名不验证** | 实际列名可能与预期不同 | 读取后用模糊匹配验证列名存在性 |
| 4 | **写入数据前不清除缓存** | 旧缓存数据导致结果不一致 | 写入前调用 `clearFileCache` |
| 5 | **对大文件一次读取所有Sheet** | 内存溢出/超时 | 按需按Sheet逐表读取 |
| 6 | **图片识别不验证文件格式** | 不支持格式导致API报错 | 检查扩展名：.png/.jpg/.jpeg/.gif/.webp/.bmp |
| 7 | **SenseNova API无重试机制** | 网络波动导致任务失败 | 最多重试3次，间隔2秒 |
| 8 | **对农户姓名/身份证号做模糊匹配替换** | 数据失真影响补贴发放 | 严格精确匹配，不自动纠错 |
| 9 | **补贴金额计算时忽略空值** | 合计金额错误 | 空值视为0参与合计，但保留空值显示 |

## 核心能力

- **自然语言查询** — 用户用口语描述需求，自动解析并提取数据
- **字段筛选** — 按列名提取指定字段（支持模糊列名匹配）
- **条件过滤** — 按条件筛选行数据（数值/文本/日期）
- **数据汇总** — 求和、平均值、计数等统计计算
- **多表处理** — 支持多Sheet和跨表数据整合
- **图片识别** — SenseNova Vision API OCR与内容理解
- **MCP Excel工具链** — 通过 `mcp__excel__*` 系列工具操作Excel

## 工作流程（7步法）

### 第1步：定位与发现数据源

**任务**：找到目标Excel文件。

用户可能提供：
- 完整文件路径（如 `D:/编程/补贴数据.xlsx`）
- 部分路径关键词（如"补贴数据""寨隆镇"）
- 未提供，需自动搜索

**未提供路径时**，按以下优先级搜索：

```
D:/编程/ 目录下的所有 .xlsx/.xls/.csv 文件
用户桌面目录
用户下载目录
工作目录
```

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| 关键词未匹配到文件 | → 列出当前目录所有 .xlsx/.xls 文件，让用户选择 |
| 找到多个候选文件 | → 展示文件列表（路径+修改时间+大小），请用户确认 |
| 目录不存在 | → 提示路径错误，请求用户提供正确路径 |
| 无权限访问 | → 提示权限不足，建议以管理员身份运行或移动文件 |

---

### 第2步：分析Excel结构

**任务**：获取Sheet列表和每个Sheet的列结构。

使用MCP工具按序执行：

```
1. mcp__excel__readSheetNames    → 获取所有Sheet名称
2. mcp__excel__analyzeExcelStructure → 获取每个Sheet的列名和数据预览
```

**输出示例**：
```
Sheet: 寨隆镇2025年地力补贴
列名: [序号, 农户姓名, 身份证号, 补贴面积(亩), 补贴金额(元), 银行卡号, 村组]
总行数: 1523
```

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| `readSheetNames` 返回空 | → 检查文件是否为有效Excel格式，提示用户确认 |
| `analyzeExcelStructure` 超时 | → 文件可能过大，改用 `readDataBySheetName` 逐Sheet分页读取 |
| 列名包含特殊字符/空格 | → 自动去除首尾空格，统一使用 `.strip()` 处理 |
| 编码乱码（中文列名显示为乱码） | → 先 `clearFileCache` 再重试，必要时用 Python pandas 指定 GBK 编码 |

---

### 第3步：解析用户需求并执行数据提取

**任务**：将自然语言需求转换为具体的数据操作。

**🔴 CHECKPOINT — 执行提取前必须确认**：
1. 目标Sheet名称正确
2. 目标列名在数据中确实存在（模糊匹配验证）
3. 筛选条件的数据类型与列类型一致

**需求 → 操作映射表**：

| 用户需求 | MCP/代码操作 |
|---------|-------------|
| "提取姓名和补贴面积" | `readDataBySheetName` → `df[['农户姓名', '补贴面积']]` |
| "找出面积大于5亩的农户" | `readDataBySheetName` → 条件筛选 |
| "计算总补贴金额" | `readDataBySheetName` → `.sum()` 统计 |
| "按村组汇总" | `readDataBySheetName` → `.groupby('村组').sum()` |
| "导出到新文件" | `writeSheetData` / `writeDataBySheetName` |

**提取代码模式**：

```python
import pandas as pd

# 基础提取
result = df[['农户姓名', '补贴面积']].copy()

# 条件筛选
result = df[df['补贴面积'] > 5]

# 分组汇总
summary = df.groupby('村组')['补贴面积'].sum()

# 复杂条件（用户描述转逻辑）
mask = (df['补贴面积'] > threshold) & (df['村组'] == village)
result = df[mask]
```

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| 列名不存在 | → 列出该Sheet所有可用列名，尝试模糊匹配（"姓名"≈"农户姓名"），确认后继续 |
| 数据类型不匹配（如"补贴面积"列为文本） | → 尝试 `pd.to_numeric(errors='coerce')` 转换，报告无法转换的行 |
| 筛选后结果为空 | → 提示"未找到符合条件的数据"，展示筛选条件供用户调整 |
| 分组聚合时列有NaN | → 根据业务规则：面积/金额NaN视为0参与合计，身份信息NaN保留 |

---

### 第4步：数据过滤与清洗

**任务**：对提取结果进行过滤、去重、格式化。

**农业补贴专项规则**：

```python
# 补贴核验表标准字段
补贴字段 = ['序号', '姓名', '身份证号', '补贴面积', '补贴金额', '银行卡号']

# 按农户提取
农户数据 = df[df['姓名'] == 农户名][['姓名', '补贴面积', '补贴金额']]

# 面积核验：合计 = sum(各丘块面积)
合计面积 = df['丘块面积'].sum()

# 产补/地力补贴分类筛选
地力补贴 = df[df['补贴类型'] == '地力补贴']
产补 = df[df['补贴类型'] == '产补']
```

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| 身份证号格式不一致（18位/15位） | → 统一转为字符串，去除空格，不做格式校验 |
| 银行卡号包含空格/分隔符 | → 自动去除所有非数字字符 |
| 重复记录 | → 按"姓名+身份证号"去重，保留第一条，报告去重数量 |
| 金额/面积列含非数字字符（如"5亩"） | → 正则提取数字部分 `re.findall(r'\d+\.?\d*', str(value))` |

---

### 第5步：图片识别（SenseNova Vision API）

**任务**：对用户提供的图片进行OCR文字提取或内容识别。

**🔴 CHECKPOINT — 调用API前必须确认**：
1. 图片文件存在且格式为 .png/.jpg/.jpeg/.gif/.webp/.bmp
2. 环境变量 `SN_VISION_API_KEY` 或 `SN_API_KEY` 已配置
3. 图片大小合理（建议 < 20MB，超大图片先压缩）

**调用方式**：

```bash
# 基本使用（输出JSON）
python scripts/sensenova_vision.py 图片路径.jpg

# 自定义提示词（农业补贴场景推荐）
python scripts/sensenova_vision.py 补贴公示照片.png --prompt "提取表格中的所有数据，包括农户姓名、补贴面积、补贴金额"

# 只输出文本（方便管道处理）
python scripts/sensenova_vision.py 图片路径.jpg --text
```

**API配置（环境变量）**：
- `SN_VISION_API_KEY`（优先）或 `SN_API_KEY`
- `SN_VISION_MODEL`（默认 `sensenova-6.7-flash-lite`）
- `SN_VISION_BASE_URL`（默认 `https://token.sensenova.cn/v1`）

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| 环境变量未配置 | → 提示用户设置 `SN_VISION_API_KEY`，给出设置方法 |
| API返回401/403 | → 提示API密钥无效或过期，请用户更新密钥 |
| API返回429（限流） | → 等待5秒后重试，最多3次 |
| API返回5xx | → 等待3秒后重试，最多3次；3次失败后保存错误日志到 `vision_error.log` |
| 图片格式不支持 | → 列出支持格式，建议用户转换格式 |
| 图片过大（>20MB） | → 用PIL压缩至 2048px 宽以内，质量85%，然后重试 |
| 返回内容无法解析为表格 | → 返回原始文本，标注"AI识别结果，请人工核对" |

---

### 第6步：结果输出与导出

**🔴 CHECKPOINT — 写入文件前必须确认**：
1. 输出路径不覆盖源文件（写新文件或新Sheet）
2. 数据已通过第4步验证（列名/类型/去重）
3. 调用 `mcp__excel__clearFileCache` 清除目标文件缓存

**输出方式决策**：

| 数据量 | 输出方式 | 工具 |
|--------|---------|------|
| < 20行 | 表格直接展示 | 终端输出 |
| 20-100行 | 表格展示 + 导出文件 | `writeSheetData` |
| > 100行 | 仅导出文件，附带统计摘要 | `writeSheetData` + 摘要 |
| 统计结果 | 卡片展示计算过程和结果 | 终端输出 |

**导出操作**：

```python
# 写入新Sheet到已有文件
mcp__excel__writeDataBySheetName(fileAbsolutePath, sheetName, data)

# 创建新Excel文件
mcp__excel__writeSheetData(fileAbsolutePath, {"Sheet1": data})
```

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| `writeDataBySheetName` 目标Sheet已存在 | → 追加时间戳后缀（如"汇总_20250602"），提示用户 |
| `writeSheetData` 路径被占用 | → 提示关闭文件后重试，或另存为带时间戳的新文件名 |
| 写入数据量过大（>10000行） | → 分批写入，每批5000行 |
| 导出路径无写权限 | → 尝试写入用户桌面或临时目录，提示权限问题 |

---

### 第7步：数据验证与核对

**任务**：对最终结果进行完整性验证。

**验证项**：
- 行数核对：源数据行数 vs 提取结果行数
- 合计校验：补贴金额/面积合计与源数据一致
- 去重校验：无重复农户记录
- 空值报告：标记空值行数

**⚠️ 失败分支**：

| 场景 | 处理方式 |
|------|---------|
| 合计金额不一致 | → 逐Sheet对比，定位差异来源，报告给用户 |
| 农户数量异常（如翻了数倍） | → 检查是否错误展开了笛卡尔积JOIN |
| 身份证号出现空值 | → 标记为"待核实"，导出时高亮标注 |

---

## 数据验证规则

**执行前验证**：
- 文件是否存在且可读
- 列名是否匹配（模糊匹配：去除空格/特殊字符后比较）
- 数据类型是否正确（数值列不应为纯文本）

**执行后验证**：
- 结果行数 ≤ 源数据行数
- 金额合计无偏差
- 关键字段（姓名/身份证号）无空值

## 农业补贴专项知识

本技能针对 **寨隆镇** 等乡镇级农业补贴核验场景深度优化：

### 补贴类型识别

| 补贴类别 | 关键列名模式 | 说明 |
|---------|------------|------|
| 地力补贴 | 补贴面积、地力、耕地 | 耕地地力保护补贴 |
| 产补 | 产补、产量、种植面积 | 粮食生产补贴 |
| 综合补贴 | 补贴金额、应发金额、实发金额 | 多种补贴合并发放 |

### 核验逻辑

```
丘块面积合计 = 补贴面积  → 核验通过
丘块面积合计 ≠ 补贴面积  → 标记差异，需人工核实
```

### 农户身份匹配

- 精确匹配：姓名 + 身份证号（去除空格后）
- 模糊匹配：仅姓名（同音字/形近字需人工确认，不自动替换）

## 注意事项

- **原始数据保留**：不修改源文件，所有操作为只读；导出写入新文件
- **小数精度**：面积保留2位小数，金额保留2位小数
- **空值处理**：显示保留空值，不替换为0；合计时空值视为0
- **编码兼容**：Windows中文环境优先GBK编码，读取失败时自动切换UTF-8
- **MCP工具链**：优先使用 `mcp__excel__*` 工具，仅在需要复杂数据处理时使用Python pandas
- **图片识别**：识别结果标注"AI识别"，关键数据需人工核对
